# 三国杀规则开发需求文档

> 来源：知乎专栏文章《三国杀规则》，原始链接：https://zhuanlan.zhihu.com/p/524905166  
> 整理日期：2026-06-08  
> 适用项目：TKGame / Unreal Engine 卡牌桌游工程  
> 文档目标：将文章中的桌游规则转化为可设计、可开发、可测试、可验收的功能需求。

## 1. 需求范围

本文档将《三国杀》规则拆分为两个玩法范围：

1. 身份基础模式：主公、忠臣、反贼、内奸身份玩法，包含开局、选将、体力、手牌、回合阶段、濒死、死亡奖惩和胜负判定。
2. 国战模式：双武将、暗将、势力、野心家、标记、鏖战、君主等规则。

建议开发优先级：

1. P0：身份基础模式核心闭环。
2. P1：武将与技能挂接框架。
3. P2：国战模式核心闭环。
4. P3：国战扩展规则，如珠联璧合、鏖战、君主规则。

## 2. 核心对象模型

### 2.1 玩家 Player

玩家应包含以下状态：

| 字段 | 类型建议 | 说明 |
| --- | --- | --- |
| PlayerId | 唯一 ID | 用于座位、回合、网络同步 |
| SeatIndex | int | 座位顺序，决定逆时针/右手边行动 |
| IsAlive | bool | 是否存活 |
| Identity | enum | 身份模式：主公、忠臣、反贼、内奸 |
| IdentityVisible | bool | 主公开局明示，其余死亡后明示 |
| GeneralCards | array | 身份模式 1 张，国战 2 张 |
| MaxHealth | int/half-unit | 体力上限 |
| CurrentHealth | int | 当前体力 |
| HandCards | array | 手牌区 |
| EquipmentZone | array | 装备区 |
| JudgementZone | array | 判定区 |
| Marks | map | 阴阳鱼、先驱、珠联璧合、野心家、鏖战等 |
| IsChainedOrStacked | bool | 国战叠置/跳过回合相关 |

### 2.2 卡牌 Card

卡牌应支持以下基础分类：

| 分类 | 说明 |
| --- | --- |
| 身份牌 | 身份模式专用，决定阵营与胜利目标 |
| 武将牌 | 提供体力、势力、性别、技能、珠联璧合关系 |
| 体力牌 | 线下表现为体力标识，程序中表现为 MaxHealth/CurrentHealth |
| 游戏牌 | 基本牌、锦囊牌、装备牌等实际可使用卡牌 |
| 标记牌 | 国战模式中用于阴阳鱼、先驱、珠联璧合、鏖战、野心家等状态 |

卡牌通用字段：

| 字段 | 类型建议 | 说明 |
| --- | --- | --- |
| CardId | 唯一 ID | 单张实例 ID |
| CardDefId | 配置 ID | 对应数据表配置 |
| Name | string | 如“杀”“桃”“酒” |
| CardType | enum | 身份、武将、基本、锦囊、装备、标记 |
| Suit | enum | 后续判定需要 |
| Rank | int/string | 后续判定需要 |
| OwnerId | PlayerId | 当前归属 |
| Zone | enum | 牌堆、弃牌堆、手牌、装备、判定、场外 |

### 2.3 游戏模式 GameMode

需要抽象模式配置：

| 字段 | 说明 |
| --- | --- |
| ModeType | Identity / NationalWar |
| PlayerCount | 玩家人数 |
| EnableGenerals | 是否启用武将 |
| EnableSkills | 是否启用技能 |
| EnableOptionalRules | 国战下家副将查看等可选规则 |
| TurnDirection | 文章规则为逆次进行 |
| InitialHandCardCount | 默认 4 |
| DrawPhaseCardCount | 默认 2 |

## 3. 身份基础模式需求

### 3.1 开局流程

系统应按以下顺序初始化对局：

1. 根据玩家人数准备身份牌。
2. 随机分配身份牌。
3. 主公立即亮出身份。
4. 其他玩家身份暗置，仅自己可见。
5. 根据配置决定是否启用武将。
6. 完成选将、体力初始化、起始手牌发放。
7. 主公作为首个行动玩家。

验收标准：

1. 主公身份在开局后对所有玩家可见。
2. 忠臣、反贼、内奸身份在死亡前只对本人可见。
3. 所有玩家开局拥有 4 张起始手牌。
4. 回合指针从主公开始。

### 3.2 身份与胜利目标

身份胜利条件：

| 身份 | 胜利目标 |
| --- | --- |
| 主公 | 杀死所有反贼和内奸 |
| 忠臣 | 保护主公，胜利条件与主公相同 |
| 反贼 | 杀死主公 |
| 内奸 | 杀死除自己外所有人 |

胜负判定规则：

1. 主公死亡时，若仅内奸存活，则内奸获胜。
2. 主公死亡时，若不是仅内奸存活，则反贼获胜。
3. 所有反贼和内奸死亡时，主公和忠臣获胜。
4. 任一胜利条件达成后，对局立即结束。

验收标准：

1. 主公死亡后系统立即结算胜负。
2. 内奸单独存活杀死主公时判定内奸胜利。
3. 反贼与内奸全部死亡时判定主忠阵营胜利。

### 3.3 选将规则

启用武将时：

1. 主公从标准包主公武将刘备、孙权、曹操加 2 张随机武将中选择 1 张。
2. 可配置为主公直接从 5 张随机武将中选择 1 张。
3. 主公选定后立即展示武将。
4. 其余玩家从随机武将中选择：
   - 10 人局以下：每人 3 张选 1 张。
   - 10 人局：每人 2 张选 1 张。
5. 其他玩家选好后同时亮出武将。

不启用武将时：

1. 每名玩家视为 4 点体力上限。
2. 每名玩家没有技能。

验收标准：

1. 主公选将先于其他玩家。
2. 非主公玩家选择过程相互不可见。
3. 所有玩家确认后才统一公开非主公武将。
4. 禁用武将时不进入选将 UI。

### 3.4 体力规则

体力初始化：

1. 玩家体力上限等于所选武将体力上限。
2. 5 人及以上局，主公体力上限额外 +1。
3. 禁用武将时，每人 4 点体力上限。

验收标准：

1. 5 人及以上身份局主公 MaxHealth 正确增加 1。
2. 当前体力初始化为体力上限。
3. 体力变化可以触发濒死与死亡流程。

### 3.5 回合流程

每名玩家回合包含 6 个阶段：

| 阶段 | 阶段名 | 默认行为 |
| --- | --- | --- |
| 1 | 准备阶段 | 触发准备阶段技能 |
| 2 | 判定阶段 | 依次处理判定区延时锦囊 |
| 3 | 摸牌阶段 | 从牌堆摸 2 张牌 |
| 4 | 出牌阶段 | 玩家使用卡牌 |
| 5 | 弃牌阶段 | 手牌数弃至当前体力值 |
| 6 | 结束阶段 | 触发结束阶段技能 |

出牌阶段限制：

1. 每个出牌阶段只能主动使用 1 张【杀】。
2. 任意角色的判定区不能存在 2 张同名牌。

验收标准：

1. 阶段按固定顺序推进。
2. 摸牌阶段默认摸 2 张。
3. 出牌阶段第二次主动使用【杀】应被系统阻止，除非后续技能或装备显式突破限制。
4. 目标判定区已有同名延时锦囊时，不能再放置同名牌。
5. 弃牌阶段结束后，手牌数不得大于当前体力值，除非存在手牌上限修正。

### 3.6 濒死与死亡

濒死流程：

1. 当角色体力降到 0 或以下时进入濒死状态。
2. 系统发起求桃/求酒流程。
3. 自己或其他玩家可使用【桃】救援。
4. 濒死角色自己可使用【酒】救援。
5. 若体力被回复到至少 1，退出濒死。
6. 若无人救援或救援不足，角色死亡。

死亡处理：

1. 死亡角色亮出身份牌。
2. 弃置其区域内所有牌。
3. 执行死亡奖惩。
4. 执行胜负判定。

死亡奖惩：

1. 任意角色杀死反贼，击杀者摸 3 张牌。
2. 主公杀死忠臣，主公弃置区域内所有牌。

验收标准：

1. 体力到 0 时不会立即移除角色，而是先进入濒死响应。
2. 死亡后身份必定公开。
3. 杀死反贼奖励摸 3 张。
4. 主公误杀忠臣时，主公手牌、装备区、判定区等区域按规则弃置。

## 4. 国战模式需求

### 4.1 开局流程

国战模式开局流程：

1. 确定起始玩家：
   - 第一局：年龄最小玩家开始。
   - 第二局起：上一轮最先阵亡玩家开始。
   - 电子化实现中应支持房主指定、随机、配置项替代年龄规则。
2. 分发武将：
   - 8 人及以下：每人 7 张武将牌。
   - 9 至 12 人：每人 5 张武将牌。
3. 玩家选择 2 张势力相同的武将。
4. 两张武将暗置。
5. 靠近体力牌者为主将，另一张为副将。
6. 初始化体力与起始手牌。

验收标准：

1. 玩家只能选择同势力双将，特殊规则另行配置。
2. 开局双将对其他玩家不可见。
3. 系统能区分主将与副将。

### 4.2 势力与暗将

暗将规则：

1. 暗置武将没有技能。
2. 暗置武将没有性别。
3. 暗置武将没有势力。
4. 只有明置至少一张武将后，角色才拥有势力。
5. 未确定势力的角色，默认不能判断为与任何角色势力相同。

明置时机：

1. 准备阶段可明置。
2. 发动武将牌技能时可明置。
3. 具有锁定技的武将在出牌阶段按规则明置。

明置结果：

1. 明置武将后确定该武将对应势力与性别。
2. 两张武将均明置后，角色性别与主将相同。
3. 第一个明置武将牌的角色获得“先驱”标记。

验收标准：

1. 暗将技能不可用。
2. 暗将不参与势力相同/不同判断。
3. 首个明置角色只获得一次先驱标记。

### 4.3 国战体力

体力上限：

1. 体力上限为两个武将完整阴阳鱼数量之和。
2. 两个单独阴阳鱼可组成 1 点体力。
3. 第一次两张武将均明置后，如存在未组成体力的单独阴阳鱼，获得 1 个阴阳鱼标记。

验收标准：

1. 双将体力计算正确。
2. 单独阴阳鱼的组合与剩余标记发放正确。
3. 当前体力随体力上限初始化。

### 4.4 野心家规则

触发条件：

1. 当一个势力的玩家数量超过总玩家数的一半时，超过阈值后该势力中新明置或符合条件的玩家成为野心家。
2. 阈值计算为玩家总数的一半向下取整。
3. 从一半开始的、之后该势力玩家均称为野心家。

野心家状态：

1. 野心家获得野心家标记。
2. 每名野心家都是独立势力。
3. 野心家之间也互为不同势力。
4. 野心家胜利目标为消灭所有其他角色。

验收标准：

1. 同势力人数达到阈值时正确转化野心家。
2. 野心家不能与原势力或其他野心家视为同势力。
3. 野心家死亡奖惩按独立势力处理。

### 4.5 国战死亡奖惩

死亡后处理：

1. 若死亡角色没有明置武将牌，需验明势力。
2. 击杀者与死亡角色同势力时，击杀者弃置所有手牌和装备牌。
3. 击杀者与死亡角色不同势力时，击杀者摸等同于死亡角色当前势力人数的牌，包含死亡角色自己。
4. 野心家杀死任意角色，摸 3 张牌。
5. 没有势力的角色杀死其他角色没有奖惩。

验收标准：

1. 死亡角色未明置时，结算前能够确定势力。
2. 同势力击杀惩罚正确执行。
3. 不同势力击杀奖励牌数等于死亡角色势力当前人数。
4. 无势力击杀无奖惩。

### 4.6 国战胜负判定

胜利条件：

1. 当全场只剩一种势力存活时，该势力角色获胜。
2. 没有确定势力的角色，即使真实武将与剩余玩家同势力，也不能获胜。
3. 野心家作为独立势力参与胜负判断。

验收标准：

1. 每次死亡后触发势力存活统计。
2. 未明置且未确定势力的存活角色不应被计入胜利阵营。
3. 野心家独自存活时判定野心家胜利。

### 4.7 珠联璧合

规则：

1. 武将配置中应声明可形成珠联璧合的武将列表。
2. 若玩家双将构成珠联璧合，在第一次两张武将均明置后获得“珠联璧合”标记。
3. 珠联璧合标记可在出牌阶段弃置并摸 2 张牌。
4. 珠联璧合标记也可当【桃】使用。

验收标准：

1. 只有配置成组合的双将获得标记。
2. 标记只在第一次双将均明置时发放一次。
3. 两种使用方式互斥，使用后标记消失。

### 4.8 鏖战模式

触发条件：

1. 游戏中仅剩 4 名及以下玩家时，检查是否进入鏖战。
2. 7 人以下游戏，仅剩 3 名及以下玩家时检查。
3. 若此时全场玩家势力均不相同，则从下一回合开始进入鏖战模式。

模式效果：

1. 场上放置“鏖战”标记。
2. 【桃】不能用于恢复体力。
3. 【桃】只能当【杀】或【闪】打出。
4. 鏖战开始后，即使后来出现两个或更多同势力角色，也不取消鏖战。

验收标准：

1. 鏖战从下一回合开始生效，不是当前检查点立即生效。
2. 鏖战状态一旦开启不会自动关闭。
3. 鏖战中使用【桃】回血应被系统阻止。
4. 鏖战中【桃】可作为【杀】或【闪】响应对应使用窗口。

### 4.9 国战标记

| 标记 | 获得时机 | 使用时机 | 效果 |
| --- | --- | --- | --- |
| 阴阳鱼 | 双将均明置且存在未组成体力的单独阴阳鱼 | 出牌阶段 | 弃置标记，摸 1 张牌 |
| 阴阳鱼 | 同上 | 弃牌阶段 | 弃置标记，本回合手牌上限 +2 |
| 先驱 | 第一个明置武将牌的角色 | 出牌阶段 | 弃置标记，将手牌补至 4 张，并观看一名其他角色的一张武将牌 |
| 珠联璧合 | 双将形成组合且首次均明置 | 出牌阶段 | 弃置标记，摸 2 张牌 |
| 珠联璧合 | 同上 | 濒死/需要桃时 | 当【桃】使用 |
| 野心家 | 满足野心家规则 | 持续 | 视为独立势力 |
| 鏖战 | 满足鏖战条件后下一回合 | 持续 | 修改【桃】规则 |

### 4.10 附加规则

可选规则：

1. 所有玩家挑选好武将后，分别观看自己下家的副将。

验收标准：

1. 该规则可通过房间配置开启或关闭。
2. 开启后只能观看下家的副将，不能观看主将或其他玩家武将。
3. 观看行为应有一次性时机限制。

### 4.11 国战君主规则

君主定义：

1. 武将牌上方势力为“君”的武将为君主武将。

替换规则：

1. 玩家第一个回合开始时，可以将两张武将牌中与君主武将同名的武将替换为君主武将。
2. 若替换导致体力上限增加，立即回复等量体力。
3. 君主武将明置后不可被暗置。

君主与野心家：

1. 君主武将明置后，所有与该君主势力相同的野心家恢复原有势力。
2. 君主死亡后，所有与其势力相同的武将都视为野心家。
3. 君主死亡后，将该君主武将置于场上作为标识，不再额外发放更多野心家牌。

验收标准：

1. 君主替换只在玩家第一个回合开始时提供。
2. 君主明置后无法重新暗置。
3. 君主明置与死亡都会重新计算相关势力/野心家状态。

## 5. 系统功能模块拆分

### 5.1 对局管理模块

职责：

1. 创建房间与对局。
2. 加载玩法配置。
3. 管理玩家座位、存活状态、回合顺序。
4. 管理游戏开始、暂停、结束、结算。

关键接口建议：

1. StartGame(ModeConfig)
2. EndGame(Result)
3. GetAlivePlayers()
4. GetNextAlivePlayer(CurrentPlayer)

### 5.2 身份与势力模块

职责：

1. 身份牌分配与可见性控制。
2. 身份模式阵营胜负判定。
3. 国战势力确定、同势力判断。
4. 野心家转化与独立势力处理。

关键接口建议：

1. AssignIdentities(PlayerCount)
2. RevealIdentity(PlayerId)
3. GetFaction(PlayerId)
4. IsSameFaction(A, B)
5. RecalculateAmbitionists()

### 5.3 武将模块

职责：

1. 武将池洗牌与分发。
2. 选将流程。
3. 主将/副将维护。
4. 明置/暗置状态。
5. 体力、性别、势力、技能入口。

关键接口建议：

1. DealGeneralCandidates(PlayerId, Count)
2. SelectGeneral(PlayerId, GeneralIds)
3. RevealGeneral(PlayerId, GeneralSlot)
4. CalculateMaxHealth(PlayerId)
5. CheckCompanionPair(PlayerId)

### 5.4 卡牌与牌区模块

职责：

1. 牌堆、弃牌堆、手牌、装备区、判定区管理。
2. 摸牌、弃牌、移动牌。
3. 判定区同名限制。
4. 死亡时区域清理。

关键接口建议：

1. ShuffleDeck()
2. DrawCards(PlayerId, Count)
3. DiscardCards(PlayerId, CardIds)
4. MoveCard(CardId, SourceZone, TargetZone)
5. HasSameNamedCardInJudgement(PlayerId, CardName)

### 5.5 回合与阶段模块

职责：

## 7. 开发里程碑

### M1：身份基础模式最小闭环

目标：

1. 支持创建 4 至 10 人身份局。
2. 支持身份分配、主公公开、起始手牌。
3. 支持 6 阶段回合流转。
4. 支持摸牌、弃牌、基础死亡、胜负判定。

不包含：

1. 完整武将技能。
2. 完整卡牌效果。
3. 国战模式。

完成标准：

1. 能从开局走到任一阵营胜利。
2. 关键状态在 UI 或调试面板可见。
3. 自动化或手动测试覆盖主公死亡、反贼全灭、内奸单独胜利。

### M2：基础卡牌与濒死响应

目标：

1. 支持【杀】使用次数限制。
2. 支持【桃】【酒】濒死救援。
3. 支持判定区同名牌限制。
4. 支持死亡奖惩。

完成标准：

1. 濒死流程可多人响应。
2. 奖惩摸牌/弃牌正确。
3. 出牌阶段限制可被测试验证。

### M3：武将框架

目标：

1. 支持身份模式选将。
2. 支持武将体力上限。
3. 提供技能触发点框架。
4. 暂不要求实现全部武将技能。

完成标准：

1. 主公与非主公选将流程符合规则。
2. 5 人及以上主公体力 +1。
3. 禁用武将配置可正常开局。

### M4：国战核心

目标：

1. 支持双将选择。
2. 支持暗置、明置、势力判断。
3. 支持国战体力计算。
4. 支持国战死亡奖惩和胜负判定。

完成标准：

1. 未明置角色无势力、无性别、无技能。
2. 势力相同/不同击杀奖惩正确。
3. 只剩一种确定势力时正确结算。

### M5：国战扩展

目标：

1. 支持野心家。
2. 支持阴阳鱼、先驱、珠联璧合标记。
3. 支持鏖战模式。
4. 支持君主规则。
5. 支持下家副将查看可选规则。

完成标准：

1. 各标记获得、使用、消耗正确。
2. 鏖战中【桃】规则正确替换。
3. 君主明置/死亡后势力与野心家状态正确刷新。

## 9. 待确认问题

1. 身份表图片未包含在正文文本中，需要补充不同人数下主公、忠臣、反贼、内奸数量。
2. 基本牌、锦囊牌、装备牌的完整牌面与效果需要单独整理。
3. 武将技能不是本文正文重点，需要独立技能需求文档。
4. “逆次进行”在电子化座位中需确认方向定义，是 SeatIndex 递增还是递减。
5. 国战“从一半开始的，之后该势力玩家都称为野心家”的精确实现需结合官方人数表校验。
6. 主公杀死忠臣时“区域内所有牌”需确认是否包含判定区；本文档按手牌、装备区、判定区均弃置处理。
7. 濒死状态下【酒】仅自己可用，是否允许非濒死使用【酒】增强伤害，需在基础牌规则文档中补充。

## 10. 非功能需求

1. 规则状态应可序列化，便于断线重连、回放和调试。
2. 关键规则结算应事件化，便于武将技能插入。
3. 客户端只提交意图，服务器或权威端负责校验合法性。
4. 所有隐藏信息需要按玩家视角过滤，避免身份、暗将、手牌泄漏。
5. 规则配置应数据驱动，避免将武将、卡牌、身份人数写死在逻辑代码中。
6. 每个阶段、响应窗口、死亡结算、胜负结算应输出调试日志。

## 11. 推荐技术落点

在 Unreal/C++ 工程中建议按以下方向实现：

1. GameMode 或 GameState 负责对局生命周期和胜负结算。
2. TurnComponent 负责回合阶段状态机。
3. CardBase/CardData 负责卡牌实例和配置。
4. PlayerState 负责玩家公开状态，PlayerController 负责私有视角与输入。
5. 使用 DataTable 或 PrimaryDataAsset 配置武将、卡牌、身份人数、模式规则。
6. 使用事件/委托系统为技能预留触发点，如 OnPhaseStart、OnCardUsed、OnDamageTaken、OnEnterDying、OnDeath、OnGameEnd。
